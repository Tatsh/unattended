The two basic principles of Windows system administration:
  • For minor problems, reboot
  • For major problems, reinstall

What is this?

This is a project for Unix systems administrators living in a Windows world. It will teach you how to boot from a floppy (or CD-ROM, or directly from the network), answer a few questions, and come back two hours later to a fully-installed Windows workstation, using nothing but Unix servers on the back end.

But be warned: This is not a ./configure && make install kind of project. This is a set of instructions, tools, and sample scripts which will require some customization for your site. Nevertheless, everything you need to know, or at least a reference to it, is here somewhere. If not, let me know; all suggestions and contributions are welcome.

Where did it come from?

In my (ever reluctant) experience with Windows, I have found that the more recently a machine was installed, the better it works. So I decided to look for ways to make the installation process routine.

This led me into the strange and quirky world of DOS networking, .bat scripts, unattend.txt files, network booting, and the amazingly under-documented silent installation procedures for various Windows applications.

Microsoft has their own solution to this problem, involving things like "Active Directory" and "RIS" and "SMS". If you think those sound like good ideas, then you are not part of this project's intended audience.

Some approaches to this problem involve drive imaging (also known as "cloning"). This project is not one of them. Workstation cloning has several drawbacks, including the SID issue, poor support for non-uniform hardware, poor maintainability, and large, bulky images. Even minor hardware variations, or minor updates to the software configuration, can cause major headaches. On the other hand, if you need to crank out a large number of completely identical workstations, drive imaging is very easy to set up. And products like Ghost and Drive Image make it very easy.

Anyway, I have assembled a complete unattended installation system from scratch, and I am releasing it here on SourceForge. Most of the information here is available elsewhere, but it took me a while to find because it was not all collected in one place. I am hoping this project will save some time for others like myself. (I would have found the database of unattended/silent installer switches for Windows apps particularly handy.)

What next?

If you are interested, you can start right now. First, visit the SourceForge project page to download the distribution. Or browse the CVS repository; just click on a revision number to view the corresponding file.

Next, read the rest of the documentation and follow the instructions. The overall process consists of two steps:

  1. Install the operating system
  2. Install the applications

Step 1 is annoying because it involves DOS networking.

Step 2 is annoying because there is no standard way to install Windows applications from the command line. Well, there is, but nobody uses it. So every application must be handled specially. Also, it is tricky to write a script which reboots the machine halfway through. (OK, the rebooting part is easy. But resuming where you left off is harder.)

Click the links above for more information.

And please let me know about any successes, failures, suggestions, or patches you have. Thanks!
