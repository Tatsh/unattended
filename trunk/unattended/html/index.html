The two basic principles of Windows system administration:
  • For minor problems, reboot
  • For major problems, reinstall

What is this?

This is a project for Unix systems administrators living in a Windows world. It will teach you how to boot from a floppy (or CD-ROM, or directly from the network), answer a few questions, and come back two hours later to a fully-installed Windows workstation, using nothing but Unix servers on the back end.

But be warned: This is not a ./configure && make install kind of project. This is a set of instructions, tools, and sample scripts which will require some customization for your site. Nevertheless, everything you need to know, or at least a reference to it, is here somewhere. If not, let me know; all suggestions and contributions are welcome.

Where did it come from?

In my (ever reluctant) experience with Windows, I have found that the more recently a machine was installed, the better it tends to work. So I decided to look for ways to make the installation process routine.

This led me into the strange and quirky world of DOS networking, .bat scripts, unattended.txt files, network booting, and the astoundingly under-documented "silent" installation procedures for various Windows applications.

Microsoft has their own solutions to these problems, involving things like "Active Directory" and "RIS" and "SMS". If you think those sound like good ideas, then you are not part of my intended audience.

Anyway, I put together a complete system from scratch and I am releasing it here on SourceForge. Most of the content here is also available elsewhere, but it took me a while to find because it was not all collected in one place. I am hoping that by creating such a place, I can save others like myself some time.

What next?

If you are interested, you can start right now. First, visit the SourceForge project page to download the distribution. (Or browse the CVS repository. Click on a revision number to view the corresponding file.)

Next, educate yourself about the overall process. It consists of two steps:

  1. Install the operating system
  2. Install the applications

Step 1 is annoying because it involves DOS network drivers.

Step 2 is annoying because there is no standard way to install Windows applications from the command line. (Well, there is, but almost nobody uses it.) So every application must be handled specially. Also, it is tricky to write a script which reboots the machine halfway through. (OK, so rebooting is pretty easy. But automatically picking up where you left off is another matter.)
