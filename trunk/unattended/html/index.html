The two basic principles of Windows system administration:
  • For minor problems, reboot
  • For major problems, reinstall

What is this?

This is a system for fully automating the installation of Windows 2000 and XP workstations, including the OS, hotfixes, and applications.

Features include:

  • Full documentation and source code
  • Support for floppy, CD-ROM, and "nothing but net" installs
  • True unattended installation, not disk imaging
  • No Windows servers required; use your Unix servers instead
  • No Unix servers required; use your Windows servers after all
  • Completely free

But be warned: This is not a ./configure && make install kind of project, and it is not a slick GUI app. You will need to understand at least a little of what goes on "under the covers", and you will need to perform some customization for your site.

Where did it come from?

In my (often reluctant) experience with Windows, I have found that the more recently a machine was installed, the better it works. So I went searching for ways to make the installation process routine.

This led me into the strange and quirky world of DOS networking, .bat scripts, unattend.txt files, network booting, and the amazingly under-documented silent installation procedures for various Windows applications.

Microsoft has their own solutions to these problems, involving things like "Active Directory" and "RIS" and "SMS". I consider those technologies are too heavyweight and complex, so this system does not depend on them.

Some approaches to these problems involve drive imaging (also known as "cloning"). This system is not one of them. Workstation cloning has several drawbacks, including the SID issue, poor support for non-uniform hardware, poor maintainability, and large, bulky images. Even small hardware variations, or small updates to the software configuration, can create large headaches. On the other hand, if you need to crank out many workstations with completely identical hardware, drive imaging is both simple and fast. And products like Ghost and Drive Image make it quite easy.

Anyway, I assembled my own complete unattended installation system from scratch, and I am releasing it here on SourceForge. Most of the information in these pages is available elsewhere, but it took me a while to find because it was not all collected in one place. I am hoping this project will save time for others like myself. (I would have found the database of unattended/silent installer switches for Windows apps particularly handy.)

What next?

If you are interested, you can start right now. First, visit the download page to download the distribution. Each release has two components: The main distribution (unattended-ver.tar.gz), and the pre-built disk images (unattended-images-ver.tar.gz). If you have a Unix machine with the right tools, you can build the disk images from the main distribution; but for now, you should probably just download both and extract them to the same folder.

Next, read the rest of this documentation. The overall process consists of two steps:

  1. Install the operating system. This step is annoying because it involves DOS networking.
  2. Install the applications. This step is annoying because there is no standard way to install Windows applications from the command line. Well, there is, but nobody uses it. So every application must be handled specially. Also, it is tricky to write a script which reboots the machine halfway through. (OK, the rebooting part is easy. But resuming where you left off is harder.)

Note that these steps describe the process which we are automating. Once you finish the preparatory work, the actual process of installing a workstation will be 1) Boot from floppy or CD or network; 2) Answer some questions; 3) Have a cup of tea.

Note that this project is still in its testing stage, so do not be surprised if you encounter problems. Please use the links on the project page to submit bug reports and feature requests. You can also browse the CVS repository; just click on a revision number to view the corresponding file.

And please let me know what you think. Thanks!
